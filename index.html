How it works

01

Point it at your project

Provide a local path to your codebase. Bakup reads your source files and recent logs in read-only mode. Nothing is indexed remotely. Nothing leaves disk.

$ bakup index --path /your/project
02

Ask what happened

Type a question in plain English. Bakup searches across your local project context to find relevant matches across code and logs.

$ bakup ask "Have we seen this null pointer before?"
03

Get an answer with citations

Responses reference the specific files, functions, and log lines that informed the answer. A confidence score is attached to every result. You decide what to act on.

> auth/session.js:142 — confidence: 0.87
04

Copy the answer into your postmortem

Every response includes the answer, confidence score, source file, and line numbers. Copy the JSON or the UI output directly into your incident timeline or runbook — sources intact.

> confidence: 0.84 · src/auth/session.py:28–45 · log:412

Your code never leaves your machine.
That's not a setting. That's the architecture.

Read-only filesystem access

Bakup requests read permissions only. It cannot create, overwrite, or delete files. There is no write path in the codebase.

No network calls during analysis

Once installed, Bakup makes zero outbound connections when processing your project. Analysis runs entirely on your hardware. Verify this with a network monitor while it runs.

No telemetry. No crash reports.

We do not collect information about how you use Bakup, what projects you point it at, or what questions you ask. We have no pipeline to receive it.

This website is the only component of Bakup that lives on a server. The tool itself has no server-side dependency.

You'll have it running before your next standup.

Docker. Four steps. Under ten minutes.

  1. 1

    Unzip, copy the env file, and set your project path. Docker handles everything else.

    # Unzip the archive
unzip bakup-developer-preview.zip && cd bakup
# Configure — set BAKUP_PROJECT_PATH to your project directory
cp .env.example .env▌
  2. 2

    Start the backend and local UI. First boot downloads the embedding model (~90 MB).

    docker compose up
  3. 3

    Index your project and an incident log. Returns a namespace you use to query.

    curl -s -X POST http://localhost:8000/index \
  -H "Content-Type: application/json" \
  -d '{"path":"/your/project","log_path":"/your/project/logs/app.log"}'
  4. 4

    Ask a question. Paste the namespace from step 3.

    curl -s -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"question":"Have we seen connection timeouts before?","namespace":"<from step 3>"}'
