# bakup.ai — Project Context

Local-first AI incident intelligence tool. Indexes codebases and log files,
then answers natural-language questions about errors, patterns, and root causes.

---

## Installer Workflow

```
User downloads bakup-ai-installer.exe  (from landing page or /download route)
  │
  ▼
Run installer  (Inno Setup, LZMA2 compressed, ~240 MB)
  │
  ├─ Extracts compiled binary to:
  │    C:\Users\<user>\AppData\Local\Programs\bakup-ai\
  │
  ├─ Installs files:
  │    bakup-launcher.bat            — entry point script
  │    bakup-server\bakup-server.exe — compiled Python backend
  │    bakup-server\ui\              — static HTML/CSS/JS
  │    bakup-server\backend\         — bundled backend modules
  │    bakup-server\_internal\       — Python runtime + wheel data
  │    bakup-server\*.dll / *.pyd    — native extensions (torch, chromadb)
  │
  ├─ Creates directories:
  │    data\vectordb\                — ChromaDB persistent storage
  │    data\model-weights\           — embedding model cache
  │
  ├─ Creates shortcuts:
  │    Start menu → bakup.ai
  │    Desktop    → bakup.ai  (checked by default)
  │
  └─ Post-install: launches bakup-launcher.bat (optional, user can skip)
```

## Startup Flow

```
bakup-launcher.bat
  │
  ├─ Sets environment variables:
  │    BAKUP_ACCESS_KEY     = tango
  │    BAKUP_HOST           = 127.0.0.1
  │    BAKUP_PORT           = 8000
  │    BAKUP_CHROMA_DIR     = <install>\data\vectordb
  │    BAKUP_MODEL_CACHE_DIR = <install>\data\model-weights
  │
  ├─ Creates data directories if missing
  │
  └─ Starts bakup-server\bakup-server.exe
```

## Launch Sequence (bakup-server.exe)

```
bakup-server.exe starts
  │
  ├─ 1. Path setup
  │    ├─ Detect frozen state (sys.frozen)
  │    ├─ Resolve APP_DIR (exe location)
  │    ├─ Create DATA_DIR alongside exe
  │    └─ Set env defaults (access key, chroma dir, model cache)
  │
  ├─ 2. Module resolution
  │    ├─ Add backend/ from sys._MEIPASS to sys.path
  │    └─ Run access key check (exits if wrong key)
  │
  ├─ 3. FastAPI app construction
  │    ├─ CORS middleware (localhost origins)
  │    ├─ Routes: /health, /index, /ask, /ask/stream, /llm/*, /debug/*
  │    └─ Mount UI static files from bakup-server/ui/ (html=True)
  │
  ├─ 4. Lifespan startup
  │    ├─ ensure_models_downloaded()  — downloads embedding model on first run (~90 MB)
  │    ├─ init_vector_store()         — initializes ChromaDB
  │    └─ Prints "bakup: ready."
  │
  ├─ 5. Browser auto-open  (background thread)
  │    ├─ Polls GET /health every 1 second
  │    ├─ Waits up to 120 seconds (first-run model download)
  │    ├─ Opens http://127.0.0.1:8000 when health returns 200
  │    └─ Fallback: opens browser after 120s even if not ready
  │
  └─ 6. Uvicorn server
       └─ Listens on 127.0.0.1:8000, blocking
```

## Build Pipeline

```
build/build.ps1
  │
  ├─ Step 1: Check prerequisites (Python, PyInstaller, Inno Setup)
  ├─ Step 2: Clean previous builds (dist/, pyibuild/)
  ├─ Step 3: PyInstaller compile (cd build && pyinstaller bakup.spec)
  │          Output: dist/bakup-server/bakup-server.exe (~44 MB exe, ~800 MB total)
  │          Post-compile: removes test/debug artifacts (~23 MB)
  ├─ Step 4: Inno Setup (ISCC.exe bakup-installer.iss)
  │          Output: dist/bakup-ai-installer.exe (~240 MB)
  └─ Step 5: Copy installer to downloads/ for website serving
```

**Build command:**
```powershell
.\build\build.ps1                      # full pipeline
.\build\build.ps1 -SkipPyInstaller     # repackage only (reuse existing binary)
.\build\build.ps1 -SkipInstaller       # compile only, no installer
.\build\build.ps1 -Verbose             # show full PyInstaller output
```

## PyInstaller Build Configuration

| Setting | Value | Rationale |
|---------|-------|-----------|
| Mode | Folder distribution (COLLECT) | One-file mode extracts to temp on every launch — slower, more AV flags |
| UPX | **Disabled** (`upx=False`) | UPX-packed binaries mimic malware packers — #1 cause of AV false positives |
| Version info | `build/version_info.txt` | Embeds CompanyName, FileDescription, ProductName — reduces AV "unknown publisher" flags |
| Console | `True` | Users see server logs in terminal window |
| Debug | `False` | No debug bootloader |
| noarchive | `False` | .pyc files bundled in archive (smaller, faster) |
| Spec file | `build/bakup.spec` | Defines hidden imports, data bundling, excludes |
| Entry point | `build/bakup_server.py` | Direct `uvicorn.run(app)` — no string import |

### Excluded modules (from spec)
- GUI: tkinter, matplotlib, PIL, IPython, jupyter
- Test: pytest, unittest, doctest
- Torch GPU: torch.cuda, torch.xpu, torch.mps, torch.mtia
- Torch dev: torch.testing, torch.distributed, torch._inductor, torch._dynamo, torch.onnx
- Dev tools: setuptools, pip, pdb, cProfile, lib2to3
- Unused: tensorboard

### Post-build cleanup (from build.ps1)
Removes directories that leak through `collect_all` despite excludes:
- `backend/.pytest_cache`, `chromadb/test`
- `torch/testing`, `torch/distributed`, `torch/_inductor`, `torch/_dynamo`
- `torch/onnx`, `torch/_export`, `torch/profiler`, `torch/package`, `torch/compiler`

### Expected binary sizes
| Artifact | Size |
|----------|------|
| `bakup-server.exe` | ~44 MB |
| `dist/bakup-server/` (total) | ~780 MB (after cleanup) |
| `bakup-ai-installer.exe` | ~240 MB (LZMA2 compressed) |

## Security Considerations

- **No UPX packing** — eliminates the most common AV false positive trigger
- **Version info embedded** — binary has proper publisher metadata (CompanyName, FileDescription)
- **Console mode** — no hidden background process; user sees server activity
- **Localhost only** — server binds to `127.0.0.1`, not `0.0.0.0`
- **No CUDA/GPU binaries** — only CPU inference; no large GPU DLLs bundled
- **Access key required** — server exits immediately if `BAKUP_ACCESS_KEY` is wrong
- **Test artifacts stripped** — no pytest/unittest code in production binary
- **Code signing** — not yet implemented; recommended before public distribution
- **Icon** — not yet added (`icon=None`); a proper .ico file improves AV trust
